# Implementation Plan: Navigation Restructure & Role-Based User Experience

**Branch**: `005-navigation-restructure` | **Date**: 2026-04-18 | **Spec**: [spec.md](file:///d:/Developpement/Projets/WEB/smart-da3m/specs/005-navigation-restructure/spec.md)
**Input**: Feature specification from `/specs/005-navigation-restructure/spec.md`

## Summary

Restructure the Ihsane platform's navigation from a disorganized mix of Stitch mockups and production routes into a clean, role-based architecture. The feature implements: (1) a public landing page at `/`, (2) a role-picker sign-in page with email+password and social OAuth (Google/Facebook), (3) student authentication via parent email + PIN, (4) complete functional dashboards for Parent (mobile-first), Student (tablet-first, age-adapted), and Expert (desktop-optimized), (5) parent registration with child onboarding and PIN generation, and (6) removal of legacy Stitch routes and spec 004 mock auth artifacts.

## Technical Context

**Language/Version**: TypeScript 5.6 (Frontend) / Python 3.11 (Backend)
**Primary Dependencies**: Vue 3.5 + Vite 6 + Pinia (Frontend) / FastAPI 0.115 + SQLAlchemy 2 + authlib (Backend)
**Storage**: PostgreSQL 16 via asyncpg
**Testing**: Vitest (Frontend) / PyTest (Backend) / Playwright (E2E)
**Target Platform**: Web SPA + RESTful API (Docker Compose)
**Project Type**: Web application (SPA + API)
**Performance Goals**: Landing page critical content < 3s, sign-in flow < 10s
**Constraints**: RTL-first, bilingual AR/FR, mobile-first (Parent), tablet-first (Student), WCAG AA
**Scale/Scope**: 3 dashboards, ~15 Vue components, ~8 API endpoints, 1 new DB model + 2 altered columns
