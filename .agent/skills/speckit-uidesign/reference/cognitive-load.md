# Cognitive Load Reference

## Three Types

### Intrinsic Load (The Task)
Manage by: breaking into steps, providing scaffolding (templates, defaults), progressive disclosure, grouping related decisions.

### Extraneous Load (Bad Design) — ELIMINATE
Sources: confusing navigation, unclear labels, visual clutter, inconsistent patterns, unnecessary steps.

### Germane Load (Learning) — SUPPORT
Via: progressive disclosure, consistent patterns, feedback, onboarding through action.

---

## 8-Item Cognitive Load Checklist

- [ ] **Single focus**: Can user complete primary task without distraction?
- [ ] **Chunking**: Information in digestible groups (≤4 items per group)?
- [ ] **Grouping**: Related items visually grouped (proximity, borders, shared background)?
- [ ] **Visual hierarchy**: Immediately clear what's most important?
- [ ] **One thing at a time**: Single decision before next?
- [ ] **Minimal choices**: ≤4 visible options at each decision point?
- [ ] **Working memory**: No cross-screen memory demands?
- [ ] **Progressive disclosure**: Complexity revealed only when needed?

**Score**: 0-1 failures = low (good). 2-3 = moderate. 4+ = critical.

---

## Working Memory Rule
Humans hold ≤4 items in working memory (Cowan, 2001).

### Practical Limits
- Navigation menus: ≤5 top-level items
- Form sections: ≤4 fields per group
- Action buttons: 1 primary, 1-2 secondary, rest in menu
- Dashboard widgets: ≤4 key metrics visible
- Pricing tiers: ≤3 options

---

## Common Cognitive Load Anti-Patterns

1. **Wall of Options**: 10+ choices, no hierarchy → Group, highlight recommended
2. **Memory Bridge**: Must remember step 1 for step 3 → Keep context visible
3. **Hidden Navigation**: Mental map required → Breadcrumbs, active states
4. **Jargon Barrier**: Technical language → Plain language, inline definitions
5. **Visual Noise Floor**: Everything same weight → Clear hierarchy
6. **Inconsistent Pattern**: Same action, different UI → Standardize
7. **Multi-Task Demand**: Simultaneous inputs → Sequence the steps
8. **Context Switch**: Jump between screens for one decision → Co-locate
