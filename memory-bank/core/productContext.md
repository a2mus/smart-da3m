# Product Context

## Target Users

### Students (الطلاب) — Primary users
| Segment | Age | Grade | Interface Needs |
|---------|-----|-------|-----------------|
| Foundation | 6–7 | السنة 1 و 2 | Audio-heavy, icon-based, minimal text, 80px+ touch targets |
| Transition | 8 | السنة 3 | French introduced, mathematical operations |
| Consolidation | 9–10 | السنة 4 و 5 | Integrative problem-solving (الوضعيات الإدماجية) |

- Auth: PIN code (4–6 digits), generated by parent
- Three visual interface variants by age band (6–7, 8–9, 10–11)

### Parents (الأولياء) — Secondary users
- Access primarily via **smartphone** (mobile-first)
- Varying digital literacy levels
- Need plain-language competency reports, not raw scores
- Auth: Email + password

### Pedagogical Experts (الخبراء) — Admin users
- Access via **desktop** (dashboard-heavy)
- Create/manage curriculum content, monitor analytics
- Auth: Email + password with RBAC roles

## Key User Flows

1. **Student Diagnostic Flow:** Login (PIN) → Select module → Take adaptive test (~10 min) → View results → Enter remediation path
2. **Student Remediation Flow:** Start session → Answer exercises (with hints) → Get immediate feedback → Pass "Passport" assessment → Advance or retry
3. **Parent Monitoring Flow:** Login → View child's radar chart → Read smart messages → Check alerts → Follow daily recommendations
4. **Expert Content Flow:** Login → Navigate curriculum tree → Create/edit module → Add questions with metadata → Preview as student → Publish
5. **Expert Analytics Flow:** Login → View competency heatmap → Auto-group students → Drill into individual profiles → Export reports

## UX Constraints
- **RTL-first** architecture (Arabic primary, French secondary)
- **CSS logical properties** for direction switching (inline-start/inline-end)
- **Bilingual** (Arabic + French) with vue-i18n
- **Mobile-first** for parent dashboard
- **Lightweight** bundle < 200 KB (future tablet ENIE deployment)
- **Touch targets** minimum 60px for student-facing elements
- **Audio support** for ages 6–8 instructions and feedback
- **No dark mode** for MVP (children need high-contrast readable interfaces)
- **Offline-capable** content caching (Workbox + Dexie.js) — full offline deferred to v2
